Galaxy Grid

Use section to initiate a new row.

To create a 12 column row, simply initiate a section. Then, add a column number from 1 to 12 on the element's g attribute to specify the width of the column.

See the Pen Galaxy Grid - Single row by Galaxy (@Galaxy) on CodePen.

Multiple rows

Sometimes you'll have a lot of rows, and nesting them in multiple sections can be verbose. This is why Galaxy columns that add up to 12 will automatically start at the next row. The example below demonstrates columns overflowing two rows:

See the Pen Galaxy Grid - Multiple Rows by Galaxy (@Galaxy) on CodePen.

Nested Columns

Each row contains 12 columns by default, but you can make a more complex layout by nesting section within other sections.

See the Pen Galaxy Grid - Nested Columns by Galaxy (@Galaxy) on CodePen.

Distributing spacing

Instead of defining equal columns, a cleaner approach can be to use Galaxy's between and around attributes on a parent section.

Use between to add equal spacing in between items.

See the Pen Galaxy Grid - Between by Galaxy (@Galaxy) on CodePen.

Use around to add equal spacing in between items and outside of it.

See the Pen Galaxy Grid - Around by Galaxy (@Galaxy) on CodePen.

Offsets

Use offset-x to push the rest of the row down.

Offsets are used to apply a distinct margins that push everything else down. Like all columns, anything that overflows 12 will start in a new row.

See the Pen Galaxy Grid - Offsets by Galaxy (@Galaxy) on CodePen.

Positioning

Use centered, left or right to position an element to its parent container.

Traditional positioning has been quite frustrating. When using Galaxy, you can take a breath of fresh air. Even vertical centering is really easy.

See the Pen Galaxy Grid - Positioning by Galaxy (@Galaxy) on CodePen.

Responsiveness

It's good practice to set your own breakpoints. This gives you full control over which components will flow and in what width.

The example below holds 4 3 columns that will span 12 columns when it reaches the md breakpoint.

See the Pen Galaxy Grid - Responsiveness by Galaxy (@Galaxy) on CodePen.

Galaxy Grid will flow columns in a new row automatically, but it's good practice to specify when exactly things will flow. To specify, just add {breakpoint}-{width} to the g tag.

Containers

Create a container to set max-widths with x-container

Containers wrap a group of components so that they don't overflow a set width. Though you can just use columns, using containers is the recommended way to maintain consistency over your design.

See the Pen Galaxy Grid - Containers by Galaxy (@Galaxy) on CodePen.
